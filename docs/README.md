# Docs Guide

This directory is split into active working docs and archived historical material.

## Active Docs

- [`docs/spec.md`](spec.md): source-of-truth requirements and safety constraints.
- [`docs/execplan.md`](execplan.md): phase-based implementation plan.
- [`docs/VALIDATION_REPORT_TEMPLATE.md`](VALIDATION_REPORT_TEMPLATE.md): delivery/validation reporting template.

## Hygiene / Context Docs

- [`docs/CONTEXT_BOUNDARIES.md`](CONTEXT_BOUNDARIES.md): what should and should not enter default AI/code context.
- [`docs/SYMBOL_MAP.md`](SYMBOL_MAP.md): lightweight module and symbol skeletons for the AI-safe code area.
- [`docs/CONTEXT_PACKS.md`](CONTEXT_PACKS.md): compact subsystem context packs.
- [`docs/REPO_INVENTORY.md`](REPO_INVENTORY.md): repo-wide inventory and cleanup signals.
- [`docs/PRODUCTION_PRUNE_PLAN.md`](PRODUCTION_PRUNE_PLAN.md): production packaging/pruning plan and release-profile guidance.

## Archived Docs

- [`docs/archive/README.md`](archive/README.md): archive purpose and contents.
- [`docs/archive/PROJECT_STATUS.md`](archive/PROJECT_STATUS.md): archived implementation snapshot.
- [`docs/archive/NEXT_STEPS.md`](archive/NEXT_STEPS.md): archived planning snapshot.

## Prompt Materials

- No tracked `prompts/` directory currently exists in the repository.
- Treat local prompt scratch files as out of default AI context unless they are intentionally added to version control later.
